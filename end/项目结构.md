## 核心模块说明

### API 层 — `api.py`

| 端点 | 方法 | 说明 |
|---|---|---|
| `/` | GET | 服务状态 |
| `/health` | GET | 进程存活检查（不依赖外部服务） |
| `/ready` | GET | 依赖就绪检查（LLM + Embedding + Chroma） |
| `/api/chat` | POST | 对话接口（非流式），自动持久化 + 画像提取 |
| `/api/chat/stream` | POST | 对话接口（SSE 流式 + 心跳保活 + 中断日志） |
| `/api/rag/search` | POST | 知识库检索（返回相关文档 + 分数 + 来源） |
| `/api/rag/status` | GET | 知识库状态（chunk 数 / 分类 / 缓存命中率 / embedding） |
| `/api/sessions` | GET | 列出所有会话 |
| `/api/session/{id}` | GET | 获取会话历史 |
| `/api/session/{id}` | DELETE | 删除会话 |
| `/api/profile` | POST | 保存/更新用户画像 |
| `/api/profile/{user_id}` | GET | 获取用户画像 |
| `/api/feedback` | POST | 提交反馈（like / dislike） |
| `/api/feedback/stats` | GET | 反馈统计 |
| `/api/admin/login` | POST | 登录获取 JWT token（HS256，24h） |
| `/api/admin/knowledge` | POST | 🔐 知识库完整结构（分类→文件→chunk） |
| `/api/admin/upload` | POST | 🔐 上传文档（multipart）并索引 |
| `/api/admin/delete` | POST | 🔐 删除文档（磁盘 + 向量库） |
| `/api/admin/category` | POST | 🔐 更改文档分类（重新索引） |
| `/api/admin/chunk/update` | POST | 🔐 编辑单个 chunk 内容 |
| `/api/admin/prompts` | POST | 🔐 提示词文件列表 |
| `/api/admin/prompt/read` | POST | 🔐 读取提示词内容 |
| `/api/admin/prompt/write` | POST | 🔐 覆写提示词文件 |

**ChatRequest 字段：** `message`、`history`、`session_id`、`user_context`、`mode`

**mode 取值：** `chat`（智能问答，默认）、`chat_only`（纯聊天）、`report`（报告生成）

**统一响应格式：** `{ success: bool, message: str, data: dict | null }`

**错误信息脱敏：** 对外返回通用文案，内部日志保留完整堆栈

### Agent 层 — `agent/react_agent.py`

- **双 Agent 架构**：
  - `self.agent`（工具模式）— 挂载全部 9 个工具 + 3 个中间件，用于 `chat` / `report`
  - `self.chat_agent`（纯聊天）— 无工具 + 仅日志中间件，用于 `chat_only`
- `execute()` — 同步调用，收集所有增量后返回完整字符串
- `execute_stream()` — 流式调用，逐块输出增量，自动过滤思考过程
- **历史压缩**：超过 15 条消息时，LLM 自动将旧消息压缩为 2-3 句摘要
- 支持 `history`（多轮对话）、`trace_id`（追踪）、`user_context`（用户上下文）、`mode`（模式选择）

### 工具层 — `agent/tools/agent_tools.py`

| 工具 | 功能 | 数据源 |
|---|---|---|
| `rag_summarize` | 知识库检索 + LLM 总结（含来源引用） | Chroma + LLM |
| `get_weather` | 获取城市真实天气 | wttr.in API，失败兜底 |
| `get_user_location` | 用户所在城市 | ip-api.com / 前端传入 / user_context |
| `get_user_id` | 用户 ID | user_context |
| `get_user_profile` | 用户完整画像（城市/性别/年龄/偏好） | SQLite 画像库 |
| `get_current_date` | 当前日期 | 系统时间 / user_context |
| `fetch_external_data` | 用户使用记录 | CSV 模拟数据 |
| `save_user_note` | 保存用户偏好/备注到画像 | SQLite 画像库 |
| `fill_context_for_report` | 触发报告模式 | 无数据（兜底方案） |

### RAG 层 — `rag/rag_service.py` + `rag/vector_store.py`

- 检索流程：**查询改写**（LLM 2-3 关键词）→ 多路检索 → 去重合并 → 关键词加权（bigram 加分）→ 阈值过滤（0.3）→ 格式化上下文 → LLM 生成
- 查询改写缓存：200 条 LRU 缓存，相同 query 不重复调用 LLM
- 缓存机制：精确匹配 + bigram 语义匹配（Overlap Coefficient ≥ 0.3），TTL 5 分钟
- 三种上下文格式：`full` / `compact` / `sources`
- 增量更新：MD5 路径组合键，自动检测文件增/删/改
- 动态 k 值：短查询 k=2，中等 k=3，长查询 k=5
- 差异化分块：按分类配置不同 chunk_size（产品FAQ=300，故障排除=400，选购指南=300）

---

### 会话层 — `utils/session_store.py`

- SQLite 数据库，存储于 `data/sessions/store.db`
- **对话消息**：session_id + role + content + 时间戳
- **用户画像**：user_id + city + gender + age + preferences(JSON) + 更新时间
- **反馈记录**：session_id + rating(like/dislike) + comment + 时间戳
- **自动画像提取**：LLM 结构化 JSON 提取优先 → 关键词规则兜底

---

### 模型层 — `model/factor.py`

- LLM：`ChatTongyi`（DashScope 通义千问），request_timeout=120s
- Embedding：`DashScopeEmbeddings`
- 懒加载单例，全局复用

## 数据流

```
请求 → api.py → ReactAgent → create_agent (LangChain)
                                  │
              ┌───────────────────┼───────────────────┐
              │                   │                   │
          工具调用             LLM 推理            RAG 检索
              │                   │                   │
        agent_tools           factor.py          rag_service
              │                   │                   │
    ┌─────────┼─────────┐     DashScope         Chroma + LLM
    │         │         │
  API调用   SQLite    CSV文件
  (天气/IP) (会话/画像) (用户记录)
```
