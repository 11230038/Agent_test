## 核心模块说明

### API 层 — `api.py`

| 端点 | 方法 | 说明 |
|---|---|---|
| `/` | GET | 服务状态 |
| `/health` | GET | 健康检查 |
| `/api/chat` | POST | 对话接口（非流式） |
| `/api/chat/stream` | POST | 对话接口（SSE 流式 + 心跳） |
| `/api/rag/search` | POST | 知识库检索 |
| `/api/rag/status` | GET | 知识库状态 |

### Agent 层 — `agent/react_agent.py`

- `ReactAgent` 类封装 LangChain `create_agent`
- `execute()` — 同步调用，收集所有增量后返回完整字符串
- `execute_stream()` — 流式调用，逐块输出增量，自动过滤思考过程（`AIMessage` 无 `tool_calls`）
- 支持 `history`（多轮对话历史）、`trace_id`（追踪）、`user_context`（用户上下文）

### 工具层 — `agent/tools/agent_tools.py`

| 工具 | 功能 | 数据源 |
|---|---|---|
| `rag_summarize` | 知识库检索 + LLM 总结 | Chroma + LLM |
| `get_weather` | 获取城市天气 | wttr.in API，失败兜底 |
| `get_user_location` | 用户所在城市 | ip-api.com / 前端传入 |
| `get_user_id` | 用户 ID | 前端传入 |
| `get_current_date` | 当前日期 | 系统时间 / 前端传入 |
| `fetch_external_data` | 用户使用记录 | CSV 模拟数据 |
| `fill_context_for_report` | 触发报告模式 | 无数据 |

### 中间件 — `agent/tools/middleware.py`

| 中间件 | 类型 | 功能 |
|---|---|---|
| `monitor_tool` | `@wrap_tool_call` | 工具执行日志 + 报告模式标记 |
| `log_before_model` | `@before_model` | 模型调用前日志 |
| `report_prompt_switch` | `@dynamic_prompt` | 动态切换报告 Prompt |

### RAG 层 — `rag/rag_service.py` + `rag/vector_store.py`

- 检索流程：Embedding 检索 → 关键词加权 → 阈值过滤（0.3）→ 格式化上下文 → LLM 生成
- 缓存机制：精确匹配 + bigram 语义匹配（Overlap Coefficient ≥ 0.3）
- 增量更新：MD5 比对，自动检测文件增删改
- 动态 k 值：短查询 k=2，长查询 k=5
- 差异化分块：按分类配置不同 chunk_size

### 模型层 — `model/factor.py`

- LLM：`ChatTongyi`（DashScope 通义千问），超时 120s
- Embedding：`DashScopeEmbeddings`
- 懒加载：首次调用时初始化，全局单例复用

## 数据流

```
请求 → api.py → ReactAgent → create_agent (LangChain)
                                  │
                    ┌─────────────┼─────────────┐
                    │             │             │
                工具调用      LLM 推理      RAG 检索
                    │             │             │
              agent_tools    factor.py    rag_service
                    │             │             │
              API / 本地     DashScope    Chroma + LLM
```
