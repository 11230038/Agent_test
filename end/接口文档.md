# 后端 API 接口文档

## 通用约定

### 响应格式

所有接口统一使用以下结构：

```json
{
  "success": true | false,
  "message": "说明文字",
  "data": { ... } | null
}
```

### 错误状态码

| 状态码 | 含义 |
|---|---|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 422 | 请求格式校验失败 |
| 502 | 上游模型服务执行失败 |
| 503 | 服务配置错误或模型不可用 |
| 500 | 服务器内部错误 |

## 接口列表

### 1. 服务状态

```
GET /
```

**响应示例：**
```json
{
  "success": true,
  "message": "ok",
  "data": { "message": "Robot Agent API is running" }
}
```

---

### 2. 健康检查

```
GET /health
```

**响应示例：**
```json
{
  "success": true,
  "message": "ok",
  "data": { "status": "ok" }
}
```

> 不依赖模型或向量库，始终可用。

---

### 3. 对话（非流式）

```
POST /api/chat
```

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `message` | string | 是 | 当前用户问题 |
| `history` | array | 否 | 历史消息 `[{role, content}, ...]` |
| `session_id` | string | 否 | 会话标识（日志追踪用） |
| `user_context` | object | 否 | 用户上下文 `{user_id, city, month}` |

**请求示例：**
```json
{
  "message": "如何更换滤网",
  "history": [
    { "role": "user", "content": "你好" },
    { "role": "assistant", "content": "你好！有什么可以帮你的？" }
  ],
  "session_id": "sess_abc123",
  "user_context": {
    "user_id": "1004",
    "city": "北京",
    "month": "2026-06"
  }
}
```

**响应示例：**
```json
{
  "success": true,
  "message": "请求成功",
  "data": {
    "answer": "滤网更换步骤如下：...参考来源：[1] 维护保养.txt",
    "trace_id": "a1b2c3d4e5f6",
    "sources": [
      { "source": "维护保养.txt", "category": "维护保养", "score": 0.714 }
    ]
  }
}
```

---

### 4. 对话（流式）

```
POST /api/chat/stream
```

**请求体：** 同 `/api/chat`

**响应格式：** SSE（Server-Sent Events）

```
data: 滤网
data: 更换步骤如下
data: ：1. 关闭
...
data:            ← 空心跳（4 秒无输出时自动发送）
data: 参考来源：[1] 维护保养.txt
```

- 内容逐增量输出
- 每 4 秒无新内容时发送空心跳保持连接
- 流自然结束即为完成

### 5. 知识库检索

```
POST /api/rag/search
```

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `query` | string | 是 | 检索关键词 |
| `style` | string | 否 | 格式化风格：`full` / `compact` / `sources`，默认 `full` |

**请求示例：**
```json
{
  "query": "WIFI设置",
  "style": "sources"
}
```

**响应示例：**
```json
{
  "success": true,
  "message": "ok",
  "data": {
    "query": "WIFI设置",
    "total": 2,
    "results": [
      {
        "content": "在APP【设置】-【消息提醒】中...",
        "score": 1.128,
        "source": "扫拖一体机器人100问.txt",
        "category": "扫拖一体",
        "metadata": { "category": "扫拖一体", "source": "..." }
      }
    ]
  }
}
```

### 6. 知识库状态

```
GET /api/rag/status
```

**响应示例：**
```json
{
  "success": true,
  "message": "ok",
  "data": {
    "knowledge_base": {
      "chunk_count": 318,
      "categories": ["产品FAQ", "故障排除", "扫拖一体", "维护保养", "选购指南"],
      "collection_name": "agent"
    },
    "cache": {
      "hits": 12,
      "misses": 45,
      "size": 8,
      "hit_rate": 0.211
    },
    "embedding": {
      "available": true,
      "error": null
    },
    "retrieval": {
      "k": 3,
      "similarity_threshold": 0.3
    }
  }
}
```

| 字段 | 说明 |
|---|---|
| `knowledge_base.chunk_count` | 向量库 chunk 总数，-1 表示异常 |
| `knowledge_base.categories` | 已索引的分类标签 |
| `cache.hit_rate` | 检索缓存命中率 |
| `embedding.available` | Embedding 模型是否可用 |
| `retrieval.similarity_threshold` | 相似度过滤阈值 |
