# 后端 API 接口文档

## 通用约定

### 响应格式

```json
{ "success": true | false, "message": "说明", "data": { ... } | null }
```

### 错误状态码

| 状态码 | 含义 | 对外文案 |
|---|---|---|
| 200 | 成功 | — |
| 400 | 请求参数错误 | 参数错误具体原因 |
| 422 | 请求格式校验失败 | "请求参数校验失败" |
| 502 | 模型执行失败 | "模型服务暂不可用，请稍后重试" |
| 503 | 配置/依赖异常 | "服务配置异常，请联系管理员" |
| 500 | 未知错误 | "服务器内部错误，请稍后重试" |

## 一、系统接口

### `GET /`
服务状态。

```json
{ "success": true, "message": "ok", "data": { "message": "Robot Agent API is running" } }
```

### `GET /health`
进程存活检查，不依赖外部服务，始终返回 200。

```json
{ "success": true, "message": "ok", "data": { "status": "ok" } }
```

### `GET /ready`
依赖就绪检查，检测 LLM + Embedding + Chroma 连通性。

```json
{
  "success": true,
  "message": "ok",
  "data": {
    "checks": {
      "llm": { "ok": true },
      "embedding": { "ok": true },
      "chroma": { "ok": true, "chunk_count": 318 }
    }
  }
}
```

有依赖异常时 `message` 会标注：`"degraded: llm, chroma 不可用"`。

## 二、对话接口

### `POST /api/chat`（非流式）

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `message` | string | 是 | 当前用户问题 |
| `history` | array | 否 | 历史消息 `[{role, content}]` |
| `session_id` | string | 否 | 会话标识，用于持久化和追踪 |
| `user_context` | object | 否 | `{user_id, city, month}` |
| `mode` | string | 否 | `chat`（默认）/ `chat_only` / `report` |

**请求示例：**
```json
{
  "message": "如何更换滤网",
  "history": [
    { "role": "user", "content": "你好" },
    { "role": "assistant", "content": "你好！有什么可以帮你的？" }
  ],
  "session_id": "sess_abc123",
  "user_context": { "user_id": "1004", "city": "北京", "month": "2026-06" },
  "mode": "chat"
}
```

**响应示例：**
```json
{
  "success": true,
  "message": "请求成功",
  "data": {
    "answer": "滤网更换步骤如下...参考来源：[1] 维护保养.txt",
    "trace_id": "a1b2c3d4e5f6",
    "sources": [
      { "source": "维护保养.txt", "category": "维护保养", "score": 0.714 }
    ]
  }
}
```

**副作用：**
- 自动保存对话到 session 存储
- 自动提取用户画像（偏好 + 性别 + 年龄）
- 自动保存 RAG 来源引用

### `POST /api/chat/stream`（SSE 流式）

请求体同 `/api/chat`，响应为 SSE 格式：

```
data: 滤网
data: 更换步骤如下
data: （空行，心跳保活）
data: ：1. 关闭电源...
```

- 4 秒无新内容时发送空心跳
- 流自然结束即完成
- 客户端断开时（AbortController / 关闭页面）后端日志记录 `[trace_id] 流式聊天被用户中断`
- 错误时发送 `data: [ERROR] 错误信息`

### mode 说明

| mode | Agent | 工具 | Prompt | 适用场景 |
|---|---|---|---|---|
| `chat` | 主 Agent | 9 个工具 | main_prompt.txt | 产品咨询、故障排查、报告 |
| `chat_only` | 聊天 Agent | 无 | chat_prompt.txt | 闲聊、打招呼 |
| `report` | 主 Agent | 9 个工具 | 自动切换 report_prompt.txt | 生成使用报告 |

## 三、知识库接口

### `POST /api/rag/search`

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `query` | string | 是 | 检索关键词 |
| `style` | string | 否 | 格式化风格：`sources`（默认）/ `full` / `compact` |

```json
{
  "success": true,
  "data": {
    "query": "WIFI设置",
    "total": 2,
    "results": [
      {
        "content": "在APP【设置】-【消息提醒】中...",
        "score": 1.128,
        "source": "扫拖一体机器人100问.txt",
        "category": "扫拖一体",
        "metadata": { "category": "扫拖一体", "source": "..." }
      }
    ]
  }
}
```

### `GET /api/rag/status`

```json
{
  "success": true,
  "data": {
    "knowledge_base": { "chunk_count": 318, "categories": [...], "collection_name": "agent" },
    "cache": { "hits": 12, "misses": 45, "size": 8, "hit_rate": 0.211 },
    "embedding": { "available": true, "error": null },
    "retrieval": { "k": 3, "similarity_threshold": 0.3 }
  }
}
```

## 四、会话接口

### `GET /api/sessions`
列出所有会话，按最后活跃时间倒序。

```json
{
  "success": true,
  "data": {
    "sessions": [
      { "session_id": "sess_abc", "msg_count": 12, "last_active": 1781246374.92 }
    ]
  }
}
```

### `GET /api/session/{session_id}`
获取指定会话的完整历史。

```json
{
  "success": true,
  "data": {
    "session_id": "sess_abc",
    "history": [
      { "role": "user", "content": "你好" },
      { "role": "assistant", "content": "你好！" }
    ]
  }
}
```

### `DELETE /api/session/{session_id}`
删除指定会话的所有消息。

## 七、管理端接口

> 先通过 `/api/admin/login` 获取 JWT token（24h 有效），后续接口传入 `token` 替代密码。

### `POST /api/admin/login` — 登录获取 JWT

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `password` | string | 是 | 管理密码 |

```json
// 请求
{ "password": "xxx" }

// 响应
{ "success": true, "message": "登录成功", "data": { "token": "eyJhbGciOiJIUzI1NiIs..." } }
```

### `POST /api/admin/knowledge` — 获取知识库结构

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `token` | string | 是 | JWT 令牌 |

返回知识库完整结构：分类 → 文件 → chunk 三层树形数据，含 chunk_id、内容预览、字符数等。

```json
{
  "success": true,
  "data": {
    "chunk_count": 226,
    "category_count": 5,
    "tracked_file_count": 7,
    "collection_name": "agent",
    "categories": [
      {
        "name": "产品FAQ",
        "file_count": 2,
        "total_chunks": 120,
        "files": [
          {
            "name": "扫地机器人100问.pdf",
            "source": "C:\\...\\data\\产品FAQ\\扫地机器人100问.pdf",
            "chunk_count": 80,
            "total_chars": 15200,
            "chunks": [
              { "chunk_id": "xxx", "content_preview": "...", "char_count": 190 }
            ]
          }
        ]
      }
    ]
  }
}
```

### `POST /api/admin/upload` — 上传文档

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `token` | string | 是 | JWT 令牌 |
| `category` | string | 否 | 分类（默认"扫地机器人客服"） |
| `file` | file | 是 | PDF 或 TXT 文件（multipart） |

同名文件自动加序号避免覆盖。上传后自动索引到向量库。

```json
{ "success": true, "message": "上传成功", "data": { "filename": "xxx.pdf", "category": "产品FAQ", "path": "..." } }
```

### `POST /api/admin/delete` — 删除文档

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `token` | string | 是 | JWT 令牌 |
| `source` | string | 是 | 文件完整路径（从 knowledge 接口获取） |

同时删除磁盘文件 + 向量库所有 chunk + MD5 记录。

### `POST /api/admin/category` — 更改文档分类

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `token` | string | 是 | JWT 令牌 |
| `source` | string | 是 | 文件完整路径 |
| `category` | string | 是 | 新分类名称 |

删除旧向量后按新分类重新索引，原文件保留在磁盘原位置。

### `POST /api/admin/chunk/update` — 编辑单个 Chunk

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `token` | string | 是 | JWT 令牌 |
| `chunk_id` | string | 是 | Chunk ID |
| `content` | string | 是 | 新内容 |

Chroma 自动重新计算 embedding。

### `POST /api/admin/prompts` — 提示词文件列表

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `token` | string | 是 | JWT 令牌 |

返回 4 个提示词文件（名称、路径、大小）。

### `POST /api/admin/prompt/read` — 读取提示词

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `token` | string | 是 | JWT 令牌 |
| `path` | string | 是 | 文件绝对路径 |

```json
{ "success": true, "data": { "filename": "main_prompt.txt", "content": "...", "char_count": 1234 } }
```

### `POST /api/admin/prompt/write` — 覆写提示词

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `token` | string | 是 | JWT 令牌 |
| `path` | string | 是 | 文件绝对路径 |
| `content` | string | 是 | 新内容 |

覆写后需重启 Agent 生效（或下次请求自动使用新 Prompt）。

### `POST /api/admin/config` — 更新配置项

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `token` | string | 是 | JWT 令牌 |
| `scope` | string | 是 | greeting/title/subtitle/placeholder |
| `mode` | string | 是 | chat/chat_only/search |
| `content` | string | 是 | 新内容 |

## 九、自定义模式接口

### `GET /api/modes` — 获取所有可用模式
返回内置 3 个 + 自定义 N 个，含 id/title/greeting/subtitle/placeholder。

### `POST /api/admin/mode/create` — 创建新模式

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `token` | string | 是 | JWT 令牌 |
| `title` | string | 是 | 模式标题 |
| `prompt_file` | file | 否 | 提示词文件（未传则 LLM 自动生成） |
| `data_files` | file[] | 否 | 数据文件（PDF/TXT） |

### `DELETE /api/admin/mode/{id}` — 删除模式
同时删除提示词文件和数据目录。内置 3 个模式不可删除。

### `POST /api/admin/mode/toggle/{id}` — 切换模式启用/禁用
内置模式状态存 `system_config(mode_enabled) → "0"/"1"`，自定义模式存 `custom_modes.enabled`。
