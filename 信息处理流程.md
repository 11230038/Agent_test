# 信息处理流程

## 总览

一条用户消息到达后端后，根据 `mode` 参数走不同路径，每条路径消耗不同数量的 LLM 调用。

| 模式 | 前端操作 | LLM 调用次数 | 适用场景 |
|---|---|---|---|
| `chat` | 智能问答 | **5**（含历史压缩则 6） | 产品咨询、故障排查、报告 |
| `chat_only` | 仅聊天 | **2** | 闲聊 |
| `search` | 知识库检索 | **0** | 直接搜索知识库 |

---

### 额外：历史压缩

当对话历史超过 15 条时，Agent 执行前额外调用一次 LLM：

```
react_agent.py:60  _compress_history()
  COMPRESS_PROMPT → LLM → 2-3句摘要
  15+ 条旧消息压缩为1条摘要消息
```

此时总调用次数 = 5 + 1 = **6**。

## 二、`chat_only` 模式（仅聊天）— 2 次 LLM 调用

```
用户消息
│
├─ [LLM 调用 ①] react_agent.py:113  chat_agent.stream()
│   chat_prompt.txt → 通义千问
│   无工具 / 无 RAG / 无 Prompt 切换
│   直接生成回复
│
└─ [LLM 调用 ②] api.py:159  画像提取（同上）
```

---

## 三、`search` 模式（知识库检索）— 0 次 LLM 调用

```
前端输入 → POST /api/rag/search
│
└─ rag_service.py:236  search(query)
   Chroma 向量检索（仅 Embedding，不调 LLM）
   返回: [{content, score, source, category}]
   
   不经过 Agent，不调 LLM
```

## 五、各调用详细信息

| 序号 | 调用位置 | 模型 | Prompt | 输入 | 输出 |
|---|---|---|---|---|---|
| ① | `react_agent.py:113` | ChatTongyi | main_prompt.txt | 用户消息 + 历史 | tool_call 或 回复 |
| ② | `rag_service.py:218` | ChatTongyi | QUERY_REWRITE_PROMPT | 原始 query | "关键词1\|关键词2\|关键词3" |
| ③ | `rag_service.py:270` | ChatTongyi | rag_summarize.txt | {context, input} | 结构化回答 |
| ④ | `react_agent.py:113` | ChatTongyi | main_prompt.txt | 工具结果 + 历史 | 最终自然语言回复 |
| ⑤ | `session_store.py:261` | ChatTongyi | EXTRACT_PROFILE_PROMPT | 对话文本 | `{gender, age, preferences}` |
| 压缩 | `react_agent.py:74` | ChatTongyi | COMPRESS_PROMPT | 旧消息列表 | 2-3 句摘要 |

## 六、缓存优化

| 缓存层 | 位置 | 命中效果 |
|---|---|---|
| 查询改写缓存 | `rag_service.py` `_rewrite_cache` (LRU 200) | 跳过 LLM 调用 ② |
| 检索缓存 | `rag_service.py` `RetrievalCache` (LRU 100, TTL 5min) | 跳过 Chroma 检索 |
| 画像关键词规则 | `session_store.py` `extract_demographics` + `extract_preferences` | LLM 失败时兜底跳过 ⑤ |
