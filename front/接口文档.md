# 前端接口消费文档

## 后端接口一览

| 接口 | 方法 | 用途 |
|---|---|---|
| `/api/chat` | POST | 智能对话 |
| `/api/rag/search` | POST | 知识库检索 |
| `/api/rag/status` | GET | 知识库状态查询 |
| `/health` | GET | 健康检查 |

---

## 1. 智能对话

### 请求

```javascript
fetch('/api/chat', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    message: '用户输入的内容',
    history: [                          // 可选，最近 15 条历史
      { role: 'user', content: '...' },
      { role: 'assistant', content: '...' },
    ],
    session_id: 'sess_abc123',          // 可选，页面级随机 ID
    user_context: {                     // 可选
      user_id: '1004',
      city: '北京',
      month: '2026-06',
    },
  }),
})
```

### 响应（成功）

```json
{
  "success": true,
  "message": "请求成功",
  "data": {
    "answer": "回答内容...",
    "trace_id": "a1b2c3d4e5f6",
    "sources": [
      { "source": "维护保养.txt", "category": "维护保养", "score": 0.714 }
    ]
  }
}
```

### 前端处理逻辑

```javascript
const data = await response.json()

// 统一错误判断
if (!response.ok || data.success === false) {
  throw new Error(data.message)
}

// 显示回答
messages.value.push({
  role: 'assistant',
  content: data.data?.answer || '暂时没有获取到回答',
})

// data.data.sources 可用于展示参考来源
// data.data.trace_id 可用于问题追踪
```

### 错误处理

```javascript
if (!response.ok) {
  // HTTP 错误（400/500/502/503）
  // data.success === false，data.message 包含可读错误信息
  // 前端直接展示 data.message 即可
}
```

## 2. 知识库检索

### 请求

```javascript
fetch('/api/rag/search', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    query: 'WIFI设置',
    style: 'sources',       // 可选：full | compact | sources
  }),
})
```

### 响应

```json
{
  "success": true,
  "message": "ok",
  "data": {
    "query": "WIFI设置",
    "total": 2,
    "results": [
      {
        "content": "在APP【设置】-【消息提醒】中...",
        "score": 1.128,
        "source": "扫拖一体机器人100问.txt",
        "category": "扫拖一体",
        "metadata": { ... }
      }
    ]
  }
}
```

### 前端渲染逻辑

```javascript
const data = await response.json()
if (!data.success) throw new Error(data.message)

searchResults.value = data.data?.results || []
// 每条结果：
//   content  → 文档片段文本
//   score    → 相似度分数（越高越相关）
//   source   → 来源文件名
//   category → 文档分类标签
```

## 3. 知识库状态

### 请求

```javascript
fetch('/api/rag/status').then(r => r.json())
```

### 响应

```json
{
  "success": true,
  "data": {
    "knowledge_base": { "chunk_count": 318, "categories": [...] },
    "cache": { "hits": 12, "misses": 45, "hit_rate": 0.211 },
    "embedding": { "available": true, "error": null }
  }
}
```

## 开发环境代理

Vite 开发服务器已配置代理，无需额外设置：

```javascript
// vite.config.js
proxy: {
  '/api': {
    target: 'http://127.0.0.1:8000',  // 后端地址
    changeOrigin: true,
  },
}
```

前端请求 `/api/chat` → 自动转发到 `http://127.0.0.1:8000/api/chat`。

## 前端状态管理

| 变量 | 类型 | 说明 |
|---|---|---|
| `messages` | `ref([])` | 完整聊天记录，页面刷新即丢失 |
| `SESSION_ID` | `const` | 页面级会话 ID，用于日志追踪 |
| `DEFAULT_USER_CONTEXT` | `const` | 默认用户信息，每次请求携带 |
| `history` | 计算值 | 取 `messages` 最近 15 条作为历史上下文 |
